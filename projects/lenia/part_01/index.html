Part 1 : ライフゲーム

November 19, 2023

0. プロジェクトの目的

Lenia は Bert Wang-Chak Chan 氏によって発表された二次元セルオートマトンです。発表されてから現在にいたるまで、多種多様なパターンが報告されています。以下に示すのはオービウムと呼ばれる基本的なパターンです。

なんとも愛らしい姿をしています。これだけでは驚きはないかもしれません。しかしオービウムは Lenia 世界における最も単純なパターンの1つでしかありません。学術誌 Complex Systems に掲載された Lenia: Biology of Artificial Life1 において、氏は次のように述べています。

Lenia は幾何学的、メタメリック2、あいまい、回復力がある、適応的、規則包括的であるという点で、他のセルオートマトンと異なる。3

Lenia と他のセルオートマトンは何が違うのでしょうか。ある意味では、セルオートマトンの自然な拡張と言えます。Bert 氏が規則包括的 (rule generic) という表現を用いているのもそのためです。人工生命の長い歴史と断絶した、突然変異種ではありません。Lenia は あくまでセルオートマトンの一般化として現れ、非常に複雑で多様なパターンを示しました。

コンピュータシミュレーションにすぎない Lenia が複雑なパターンを示し、ときに生命のような存在として振舞う。どんな仕組みがそれを可能にするのでしょうか。疑問は尽きません。そこで、筆者は Go 言語で Lenia のシミュレーションを行うことにしました。先人の知恵に頼りながら、自分で作ることによって学ぶというわけです。

私自身、このプロジェクトを進める過程で Bert 氏のチュートリアル4から多くを学びました。これがなければ、自分で別言語で実装してみよう、などとは決して思わなかったでしょう。読者の人工生命への旅にお役に立てれば幸いです。

まずは Lenia への第一歩として、ライフゲームについておさらいします。

1. ライフゲームのの概要

1.1. 導入

漫画家の士郎正宗氏は『攻殻機動隊』を描くことになったきっかけについて、2023年10月31日に公開されたインタビュー5の中で以下のように答えています。

1970年冬、『ライフゲイム』という放置観察系コンピューターゲーム、あるいは自動描画アートの一種が世に紹介されました。のちにこれに刺激を受けて1980年代初期に物語世界観の骨格をつくったのです

放置観察系コンピューターゲームとは何とも的確な表現です。攻殻機動隊の壮大な世界に影響を与えたライフゲームとはいったい何なのでしょうか。

1.2. 定義

ライフゲーム（Conway’s Game of Life6）は二次元グリッド上のセルオートマトンです。グリッド上に配置された各セルはオンとオフ、2つの状態をとります。二次元の配列に 0 か 1 が格納されている状態を想像するとわかりやすいと思います。セルの状態はある規則にしたがって更新されていき、刻々と変化する様子が興味深いパターンを示します。

グリッドを表現するにあたって、int 型のスライスを定義します。stateは現在のグリッドの状態、nextStateは次のグリッドの状態を表します。グリッドはrows * cols個のセルで表現されます。

type World struct {
	(省略)
	rows int
	cols int
	(省略)
	state [][]int
	nextState [][]int
}

グリッドはランダムに初期化するか、特定のパターンを読み込むか、いずれかの方法をとります。以下は乱数で初期化するプログラムです。

func (w *World) init(randomInit bool){	
	w.state = make([][]int, w.rows)
	w.nextState = make([][]int, w.rows)
	for i:=0;i<w.rows;i++ {
		w.state[i] = make([]int, w.cols)
		w.nextState[i] = make([]int, w.cols)
	}
	if randomInit {
		s1 := rand.NewSource(time.Now().UnixNano())
		r1 := rand.New(s1)
		for i:=0;i<w.rows;i++ {
			for j:=0;j<w.cols;j++ {
				w.state[i][j] = r1.Intn(2)
				w.nextState[i][j] = w.state[i][j]
			}
		}
	}
}

randomInitは乱数で初期化するかどうかを決める変数で、この変数がfalseの場合はすべてのセルがオフ状態で初期化されます。

2. 近傍

2.1. 近傍とは

セルを更新する規則とは何でしょうか。セルの次の状態は、そのセル自身の周りにあるオン状態のセルの個数に依存します。セルに隣接しているのは東西南北4つのセルで、斜めのセルも含めると合計8つのセルが周辺に存在することになります。そのような8つのセルのことを今後は近傍(Neighborhood7)と呼びます。近傍のオン状態のセルの個数を数えることは難しくありません。

先ほど定義したstateのうち(x,y)という座標にあるセルはstate[y][x]で表せます。このセルの近傍のオン状態のセルを数えたいのであれば、次のような処理を実行すれば十分です。getState(y,x)は座標に基づいてセルの状態を取得します。

func (w *World) neighbor(y, x int) int {
	count := 0
	for j:=-1;j<=1;j++ {
		for i:=-1;i<=1;i++ {
			if i!=0 || j!=0 {
				count += w.getState(y+j,x+i)
			}
		}
	}
	return count
}

ただし、一つ問題があります。グリッド自体は有限のサイズですから、端にあるセルの近傍をどう定義するのかということです。

2.3.1. ゼロ埋め

$a_{00}$に対して、どのように近傍を定義すべきでしょうか。1つ目の考えは、グリッドの周辺にオフの状態のセルがあるというものです。

こうすれば周辺に8つのセルがあると見なすことができます。

2.3.2. ドラクエ世界、あるいはドーナツワールド

2つ目の考えは、グリッドの端と端が繋がっているというものです。

ドーナツの表面のような、上に行っても右に行ってもどこまでも終わりがない世界を想像すると良いと思います。セルの近傍については以上2つの考え方がわかっていれば問題ありません。

2.4. プログラム

これまで解説した近傍の考え方に沿ってgetState()を実装すると以下のように書くことができます。wrapはグリッドからはみ出した場合にゼロ埋めするかどうかを決める変数です。ゼロ埋めしない場合はドーナツ型のグリッドであると仮定してセルの状態を返します。

func (w *World) getState(i,j int) int {
	if w.wrap {
		for i<0 {
			i+=w.rows
		}
		for i>=w.rows {
			i-=w.rows
		}
		for j<0 {
			j+=w.cols
		}
		for j>=w.cols {
			j-=w.cols
		}
		return w.state[i][j]
	} else {
		if i<0 || i>=w.rows || j<0 || j>=w.cols {
			return 0
		}else{
			return w.state[i][j]
		}
	}
}

セルの近傍についてはここまでにして、セルの状態を更新する規則を見ていきましょう。

3. 規則

3.1. 更新規則

セルの更新規則は以下の3つ（4つ？）です。オン状態のセルが過密すぎても過疎すぎてもいけないということをうまく定式化していると言えます。

規則1

あるセルがオン状態のとき、その近傍のオン状態のセルが4つ以上もしくは2つ以下ならば、そのセルはオフになる

規則2

あるセルがオン状態のとき、その近傍のオン状態のセルが2つもしくは3つならば、そのセルはオンのままである

規則3

あるセルがオフ状態のとき、その近傍のオン状態のセルがちょうど3つならば、そのセルはオンになる

3.2. プログラム

各セルを規則1~3にしたがって更新するプログラムは以下の通りです。すべてのセルについて次の状態が計算できたら、copyによってnextStateをstateへコピーします。

func (w *World) update() {
	for y:=0;y<w.rows;y++ {
		for x:=0;x<w.cols;x++ {
			s := w.neighbor(y,x)
			if w.state[y][x]==1 {
				if s!=2 && s!=3 {
					w.nextState[y][x] = 0
				}
			}else{
				if s==3 {
					w.nextState[y][x] = 1
				}
			}
		}
	}
	for y:=0;y<w.rows;y++ {
		copy(w.state[y], w.nextState[y])
	}
	w.generation++
}

5. おわりに

Leniaを理解するために本稿ではライフゲームについておさらいしました。プログラムは GitHub8 にて公開しています。

今回触れられなかったトピックとしてビット演算があります。これについては Michael Abrash 氏による教科書9の17章が参考になると思います。またグリッドを効率良く更新する方法についても紹介できませんでした。興味がある方は Tony Finch 氏のサイト10を参考にしてください。Go 言語による実装としては sachaos 氏による go-life11 があります。ライフゲームの全容を知りたい方には Nathaniel Johnston 氏によって運営されている ConwayLife.com12 をおすすめします。総本山ともいえるようなサイトです。また、ライフゲームの世界をもっと体験したい方は Golly13 を使うのが有力な選択肢です。

Leniaへの旅は始まったばかりです。最後までお付き合いいただけると幸いです。
